Types of SQL

Data Definition Language (DDL) - commands to create database objects such as tables, indexes, and views.

  • CREATE

  • ALTER

  • DROP


Data manipulation Language (DML) – commands toquerying and modify data.

  • SELECT

  • INSERT

  • UPDATE

  • DELETE


Transaction control language (TCL) - commands are used to manage transactions in the database.

  • BEGIN

  • TRAN

  • COMMIT

  • ROLLBACK


Data control language (DCL) – commands for security permission in a database.

  • GRANT

  • REVOKE


DDL

    o Stands for Data Definition Language.

    o DDL commands to create the structure of the database and the other database objects.

    o Its commands are auto-committed.

    o The changes are saved in the database permanently.


Truncate

    o DDL Command

    o Executed using table lock. Whole table is locked while removing the records.

    o Removes all rows from a table.

    o Minimal logging in transaction log, so it is performance wise faster

    o Cannot be rolled back


Drop

    o DDL Command

    o DROP command removes a table from database

    o Tables rows, indexes and privileges will also be removed.

    o It slower than TRUNCATE.

    o Operation cannot be rolled back.


DML

    o Stands for Data Manipulation Language.

    o Manage the data stored in the database.

    o Not auto-committed.

    o Changes are not permanent.

    o It is possible to roll back the operation.


TCL

  • To manage transactions in the database

DCL

    § A syntax similar to a computer programming language used tocontrol access to data stored in a database (Authorization)
